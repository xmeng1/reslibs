# 项目使命

### 需求: ResLibs 通用资源平台使命
系统应提供一个自动化的多类型资源分享平台，实现从源存储到变现存储的内容聚合和分发，支持包括但不限于 Unity Assets、软件工具、设计素材、教育课程等多种资源类型。

#### 场景: 平台目标定义
- **当** 利益相关者审查项目使命时
- **那么** 他们理解平台支持通过 `reslibs.com` 进行多类型资源聚合和分发

#### 场景: 多类型资源支持
- **当** 平台处理不同类型资源时
- **那么** 它自动适配并支持：Unity Assets、3D 模型、设计素材、软件工具、文档资料、视频课程、音频资源等

#### 场景: 核心价值主张
- **当** 评估平台价值时
- **那么** 它提供：通用化的资源检索界面、可配置的资源类型管理、简化的管理员发布流程、通过高级文件托管服务实现的变现

#### 场景: 商业模式清晰度
- **当** 审查商业策略时
- **那么** 收入通过诚通/飞猫/Rapidgator 下载佣金和网站广告产生，支持不同资源类型的差异化变现策略

### 需求: 核心功能模块
系统应实现三个核心模块以完成完整的资源处理工作流，支持多种资源类型。

#### 场景: Web 前端模块
- **当** 用户与平台交互时
- **那么** 他们可以访问：按资源类型筛选的资源列表、适配不同资源的详情页面、智能搜索功能、多维度分类浏览

#### 场景: Web 后端模块
- **当** 管理员管理内容时
- **那么** 他们使用 CMS 管理：多类型资源数据、可配置的元数据字段、用户权限、资源类型定义

#### 场景: 自动化机器人模块
- **当** 处理新资源时
- **那么** 自动机器人执行：多源文件下载（百度网盘等）、智能文件识别和分类、资源解压和元数据解析、基于资源类型的 LLM 内容增强、多主机分发、自动化 CMS 发布

### 需求: 资源类型扩展性
系统应支持灵活添加新的资源类型，无需核心代码修改。

#### 场景: 动态资源类型定义
- **当** 管理员添加新资源类型时
- **那么** 系统支持：自定义元数据字段、资源特定的预览和展示模板、类型特定的处理规则

#### 场景: 资源识别自动化
- **当** 系统处理未知类型文件时
- **那么** 它基于文件扩展名、内容特征和用户定义规则自动识别资源类型并应用相应处理流程

#### 场景: 特化处理适配
- **当** 系统处理特定资源类型时
- **那么** 它应用：类型特定的解压和处理逻辑、定制化的内容生成模板、特定的分发策略
