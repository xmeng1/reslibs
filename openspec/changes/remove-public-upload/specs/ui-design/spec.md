## MODIFIED Requirements
### Requirement: 多类型资源布局架构
系统应实现干净、基于网格的布局，支持多种资源类型的展示，专注于用户浏览和搜索体验。

#### Scenario: 顶部导航
- **当** 用户查看网站头部时
- **那么** 他们看到：左对齐的徽标（"ResLibs"）、居中的醒目圆角搜索栏、资源类型选择器、右对齐的分类链接（按资源类型分组）、管理后台入口（管理员可见）

#### Scenario: 简化的主页布局
- **当** 用户访问主页时
- **那么** 系统显示：突出的搜索栏、精选资源列表、资源类型筛选器，移除所有公共上传相关UI元素

#### Scenario: 专注的搜索体验
- **当** 用户搜索资源时
- **那么** 系统提供：居中突出的搜索栏、类型感知的搜索建议、多维度筛选（类型、分类、标签）、搜索结果高亮

## ADDED Requirements
### Requirement: 管理员专用上传功能
系统应提供仅限管理员使用的资源上传功能，确保内容质量和平台定位。

#### Scenario: 管理员上传入口
- **当** 管理员用户访问网站时
- **那么** 他们可以看到管理后台入口，用于资源管理和上传操作

#### Scenario: 公共用户上传限制
- **当** 普通用户访问网站时
- **那么** 他们看不到任何公共上传入口，只能浏览和搜索现有资源
